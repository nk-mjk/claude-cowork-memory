# Cowork-Memory

Claude Cowork 세션 간 지식을 누적·공유하기 위한 경량 위키 시스템입니다.

---

## 왜 필요한가?

Claude Cowork는 대화(세션)가 끝나면 맥락이 초기화됩니다. 새 대화를 시작할 때마다 "지난번에 뭘 했더라", "내 스타일 또 설명해야 하나"를 반복해야 하죠.

Wiki 시스템은 이 문제를 두 가지 방식으로 해결합니다.

- **about_me.md** — 나에 대한 기본 정보를 저장해두면, Claude가 매 대화마다 자동으로 읽고 나를 기억한 채로 시작합니다.
- **지식 노트** — 세션마다 새로운 결정·발견을 노트로 남기면, 다음 세션에서 이어받을 수 있습니다.

---

## 작동 원리

```
매 대화 시작 시                      매 대화 마무리 시
        ↓                                   ↓
INDEX.md 읽기                 새 지식·결정 → 노트 작성 + INDEX 갱신
        ↓                                   ↓
* 표시 파일 즉시 로딩           다음 세션의 Claude가 이어받음
(about_me.md 등)
        ↓
지식 노트 목록 파악
(필요할 때 열어봄)
```

---

## 파일 구조

```
your-cowork-folder/
├── CLAUDE.md                    ← Claude 행동 설정
└── wiki/
    ├── INDEX.md                 ← 지식 지도 (항상 로딩 목록 + 지식 노트 목록)
    ├── about_me.md              ← * 항상 로딩: 사용자 정보, 말투, 작업 스타일
    ├── 20260101_첫번째주제.md   ← 개별 지식 노트
    └── 20260102_두번째주제.md
```

## 설치 방법

### 1단계 — 폴더 구조 생성

코워크 작업 폴더 안에 `wiki/` 폴더를 만들고 이 레포의 파일들을 복사합니다.

```
your-cowork-folder/
└── wiki/
    ├── INDEX.md
    └── about_me.md
```

### 2단계 — about_me.md 작성

`wiki/about_me.md`를 열고, 샘플 내용을 자신에게 맞게 수정합니다. 직접 파일을 편집해도 되고, Claude에게 나에 대한 이야기를 나눈 뒤 수정을 요청해도 됩니다.

Claude가 매 대화마다 나를 알아줬으면 하는 내용을 자유롭게 넣으면 됩니다. 이름, 직업, 선호하는 말투, 작업 스타일, Claude와 어떻게 협업하고 싶은지 등을 작성하세요.

### 3단계 — CLAUDE.md 설정

이 레포의 `CLAUDE.md` 파일을 사용합니다.

- **CLAUDE.md가 없는 경우** → 이 레포의 `CLAUDE.md`를 작업 폴더에 그대로 복사하면 됩니다.
- **이미 CLAUDE.md가 있는 경우** → `CLAUDE.md`의 내용을 기존 파일에 붙여넣습니다.

---

## INDEX.md 구조

INDEX.md는 두 섹션으로 구성됩니다.

```markdown
## * 항상 로딩

- * [[about_me]] — 사용자 정보, 선호 말투, 작업 스타일

## 지식 노트

- [[20260101_주제]] — 한 줄 핵심 요약
- [[20260102_주제]] — 한 줄 핵심 요약
```

**`*` 항상 로딩**: 매 대화 시작 시 자동으로 읽히는 파일입니다. `about_me.md` 외에도 항상 기억해야 할 파일이 생기면 이 섹션에 `* [[파일명]]` 형식으로 추가하면 됩니다.

**지식 노트**: 세션별 지식 목록입니다. Claude가 관련 주제가 나올 때 열어봅니다.

## 지식 노트 작성 규칙

### 노트 작성 트리거

- 의미 있는 작업 단위가 완료됐을 때 → Claude가 "이 내용을 위키 노트로 남겨둘까요?" 라고 먼저 제안합니다.
- 사용자가 "마무리", "정리해줘", "노트 남겨줘" 등 명시적으로 요청할 때 → 즉시 작성합니다.
- 단순 작업·일회성 대화라면 → 노트를 만들지 않습니다.

### 파일명
```
YYYYMMDD_주제설명.md
```
예: `20260622_API연동방법.md`

### 내용 구조
```markdown
# 주제 제목

> 한 줄 요약 (INDEX.md에 들어갈 문장)

## 핵심 내용

이 세션에서 발견한 지식·결정 사항을 기록합니다.

## 관련 노트

- [[20260101_관련주제]] — 연결 이유
```

### 한 줄 요약 작성 팁

"무엇을 했다"보다 "무엇을 발견했다 / 결정했다"로 씁니다. 다음 세션의 Claude가 관련 노트를 열어볼지 판단하는 근거가 됩니다.

---

## 노트 간 연결

노트 안에서 다른 노트를 `[[파일명]]` 형식으로 참조할 수 있습니다 (확장자 제외).

```markdown
이 결정은 [[20260601_초기설계]] 에서 세운 원칙을 따릅니다.
```

존재하지 않는 노트를 참조해도 괜찮습니다. "나중에 만들어야 할 노트"를 미리 표시하는 용도로도 씁니다.

## 권한 구조

| 작업 | 허가 필요 여부 |
|------|--------------|
| 새 지식 노트 추가 | 자동 (허가 불필요) |
| INDEX.md 갱신 | 자동 (허가 불필요) |
| 기존 노트 내용 수정 | 사용자 허가 필요 |
| 기존 노트 삭제 | 사용자 허가 필요 |
| about_me.md 수정 | 사용자 허가 필요 |
