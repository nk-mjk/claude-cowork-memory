# 노트 작성 예시 — API 연동 방식 결정

> Kakao API 연동은 서버사이드 프록시 방식으로 결정. 클라이언트 직접 호출 시 키 노출 위험.

## 핵심 결정

- 클라이언트에서 Kakao API를 직접 호출하면 API 키가 노출될 수 있음
- 서버사이드 프록시를 두고 클라이언트는 내부 엔드포인트만 호출하는 방식으로 확정
- 관련 코드는 `src/api/kakao-proxy.js`에 작성

## 발견한 제약 사항

- Kakao API 무료 플랜 기준 일 1,000회 호출 제한
- 응답 캐싱 필요 — Redis 도입 검토 중 (미결)

- 에러 핸들링 패턴 통일
